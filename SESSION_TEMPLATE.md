# Growee — Session Template

Стартер нового чата для этого репо. Работает одинаково для Claude Code и Codex — просто скопируй нужные блоки в первое сообщение сессии.

Парные файлы:
- `CLAUDE.md` — статика проекта (Claude-слой)
- `AGENTS.md` — статика проекта (Codex-слой)
- `pipeline.md` — динамика, точки входа, worked examples
- `implementation-plan.md` — roadmap по чанкам (если задача крупная)

## Session Starter

```text
Работаем только в рамках проекта Growee.

Правила на эту сессию:
- Сначала изучи релевантные файлы (CLAUDE.md / AGENTS.md / pipeline.md), потом меняй код.
- Не делай лишний рефакторинг вне задачи.
- Сохраняй существующий архитектурный паттерн.
- Для команд, связанных с рантаймом проекта, используй project-level wrapper / preferred команды.
- Не читай `.env`, `.env.*`, `.env.backup` и другие секреты.
- Если задача большая, сначала выдели chunk и выполни его до конца.
- Если меняешь runtime-логику или маршрутизацию, проверь конфиги, pipeline.md и нужные prompt/template файлы.
- Ответы пиши кратко и по-русски.
```

## Optional Add-Ons

Добавляй когда уместно.

### Safe Editing

```text
- Меняй только файлы, которые реально нужны для этой задачи.
- Не трогай пользовательские изменения вне своей области.
- Перед завершением проверь, нет ли очевидной регрессии.
```

### Implementation Mode

```text
- Не останавливайся на анализе: если решение понятно, сразу вноси изменения.
- После правок сделай быструю проверку и дай короткий итог по результату.
```

### Planning Mode

```text
- Сначала сформулируй короткий план chunk'ами.
- Выполняй только текущий chunk, не расползайся на соседние задачи.
```

### Review Mode

```text
- Смотри на задачу как code review.
- Сначала перечисли риски, баги и регрессии, потом уже summary.
```

## When To Use

- Новая задача в этом репо
- Новая сессия после `/clear` или контекст-резета
- Большое изменение, где хочется явно сузить scope
